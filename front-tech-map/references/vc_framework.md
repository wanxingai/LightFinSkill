# VC 分析框架

> 技术图谱的最终用户是 **VC 技术尽调**，图谱必须回答以下问题。

## A. 创始人技术深度评估

- 创始人是该技术方向的**核心人员**还是**边缘人员**？
- 其核心技术贡献是**独立发明**还是**团队/导师成果的延伸**？
- 在图谱上的表现：核心人员 = 粗边框节点 + 多条上游连线；边缘人员 = 细边框 + 孤立

## B. 技术路线区别度

- 不同流派的**技术差异**是什么？（模拟 vs 数字 vs 混合、SNN vs ANN+SNN 融合等）
- 在图谱上的表现：技术概念节点(紫色)标注核心差异 + 各流派 cluster 不同底色

## C. 中外领先度判断

- 国内公司是海外技术的**追随者/继承者**（= 上游连线指向海外导师），还是**自主培养**（上游全部为国内节点）？
- 领先度指标：技术节点密度（某流派节点 > 另一流派 → 生态更成熟）
- 在图谱上的表现：中国节点橙/粉色，海外节点蓝/绿色，连线方向和密度直接可读

## D. 技术可追溯性（Defensibility）

- 公司的技术护城河能追溯到哪一层？
- 追溯层数越多（PhD → 博后 → 实验室 → 公司）= 技术积累越深
- 追溯断裂（孤儿节点）= 技术来源不明 = 潜在风险信号

## VC 分析标注（每个公司节点必须包含）

| 🏷️ 标签 | 含义 | 判据 |
|---------|------|------|
| `🏷️ 继承` | 创始人继承导师/实验室技术 | 上游有明确学术连线 |
| `🏷️ 自主` | 国内自主培养，无海外源头 | 上游全部为国内节点 |
| `🏷️ 独立` | 创始人独立发明，无学术溯源 | 无PhD/无导师，独立专利 |
| `🏷️ 回归` | 创始人从其他领域回归 | 之前做的不是本领域核心 |
| `⚠️ 非核心` | 公司产品偏离图谱核心领域 | 如：DL加速≠类脑 |
